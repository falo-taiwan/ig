COLLABORATION THINKING

1. 選擇不對的協作方法，事倍功半

當我們把「精密運算」的任務強加給「語意對話」的工具時，再優秀的 AI 夥伴也只能給出令人挫折的答案。

真實痛點情境

「我只是要 AI 比對兩個 Excel 裡同名工作表的資料，看專案是否有新增、刪減或欄位變異。結果它一直抓錯，我重新下指令說關鍵字一樣就算同個專案，它還是錯。我搞了一個多小時，到最後我自己人工花半小時搞定。AI 真的很難用！」

某企業專案經理 (PM) 因為「選錯協作模式」而感到挫折的典型使用者

協作方法的本質誤區

這位經理的困境，不在於 AI 的智商高低，而在於「誤用了 AI 的協作模式」。直接把檔案丟給聊天對話框（Chat Mode），是強迫一個極具創造力的「文學家」去幹精密的「會計審計」工作。

  • 聊天對話（Chat）的特性：擅長邏輯推理、概念擴展、語意摘要。但具有隨機性（Probabilistic）。
  • 程式代碼（Code）的特性：擅長絕對精確、重複性高、邏輯嚴密的資料對齊。具有決定性（Deterministic）。
  • 最佳協作法：人出點子 ➡️ AI 寫出程式 ➡️ 程式精確執行。這才是「選對方法」的黃金三角。

METHODOLOGY BOUNDARY

3. 協作方法論：認識「對話框」與「程式」的局限

為什麼直接把資料餵給 AI 對話框，它無法給你 100% 的精確度保證？這與模型的底層運算邏輯息息相關：

1. 機率預測非硬性比對

AI 的本質是「預測下一個最可能出現的字（機率）」，而不是資料庫的精確雜湊（Hashing）匹配。它擅長讀懂語意，但不擅長做數學上的等號校對。

2. 注意力稀釋與滑動

當 30-50 列資料長得極度重覆時，AI 的 Attention 權重會被高度均勻稀釋。在掃描過程中容易發生「注意力滑動」，誤將 A 專案的資料貼給了相鄰的 B 專案。

3. 「迷失在中間」效應

Transformer 模型對輸入開頭與結尾的抓取精度極高（如 PRJ-001 與 PRJ-050），但位於「長文本中間」的 30 筆資料，其提取權重會大幅衰減，導致漏看。

4. 工作記憶體定址上限

要精確對齊 500 個單元格，AI 必須在單次思考中維護一個極為龐大的變數定址映射表。這超出了 LLM 單次前向傳播（Forward Pass）的短期工作記憶極限。

6. 標準解決方案：企業級 Python 比對程式碼

要達到 100% 絕對精度，唯一正確做法是利用 AI 寫出精準的程式來運行。以下是經受實戰檢驗的標準 Python 比對範本：

compare_excel.py 
import pandas as pd
import openpyxl
from openpyxl.styles import PatternFill, Font, Alignment, Border, Side

def compare_sheets(file_old, file_new, sheet_name, key_col):
    # 1. 載入並去除主鍵的頭尾空白 (Trim Keys)
    df_old = pd.read_excel(file_old, sheet_name=sheet_name)
    df_new = pd.read_excel(file_new, sheet_name=sheet_name)
    
    df_old[key_col] = df_old[key_col].astype(str).str.strip()
    df_new[key_col] = df_new[key_col].astype(str).str.strip()
    
    # 2. 轉換為字典進行極速對齊
    dict_old = df_old.set_index(key_col).to_dict(orient='index')
    dict_new = df_new.set_index(key_col).to_dict(orient='index')
    
    keys_old, keys_new = set(dict_old.keys()), set(dict_new.keys())
    
    # 3. 找出新增、刪除、與交集專案
    added = sorted(list(keys_new - keys_old))
    deleted = sorted(list(keys_old - keys_new))
    common = keys_old & keys_new
    
    # 4. 逐欄比對數值異動
    modified = []
    for key in sorted(list(common)):
        chg = {}
        for col in [c for c in df_old.columns if c != key_col]:
            v_old, v_new = dict_old[key].get(col), dict_new[key].get(col)
            if pd.isna(v_old) and pd.isna(v_new): continue
            if v_old != v_new:
                chg[col] = {"old": v_old, "new": v_new}
        if chg:
            modified.append({"key": key, "changes": chg, "row_new": dict_new[key]})
            
    print(f"比對完成! 新增: {len(added)} 筆, 刪除: {len(deleted)} 筆, 變更: {len(modified)} 筆。")
    return added, deleted, modified

# 呼叫範例：

「把『模糊、有邏輯溫度』的指令留給人機溝通；把『精確、重覆性高』的運算留給程式。而 AI 的職責，是當你的工程師，幫你把『模糊指令』翻譯成『精確的程式碼』去執行！」
