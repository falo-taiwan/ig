Sunny廚苑 - 訂餐系統 POC v1.0

基礎單語前後端串接概念驗證版 (Concept of Proof)

1. 整體設計邏輯

v1.0 版本旨在快速建立一個具備前後端分離概念的訂餐 POC 系統。 前端介面使用純 HTML5/CSS3/JavaScript 構建，後端使用 Google Apps Script (GAS) 將數據存入 Google 試算表（Spreadsheet）中。

核心交互流程：

  1. 商品加載：前端加載時，呼叫後端的 getProducts()，讀取試算表中的菜單並動態繪製成卡片。
  2. 購物車機制：使用者點選商品後加進購物車，由 JavaScript 在記憶體中維護訂餐籃列表，並即時更新合計金額。
  3. 送出訂單：使用者填寫收件姓名、電話、取餐方式與備註，點擊送出後將整筆訂單物件打包，呼叫後端 submitOrder(orderData) 寫入試算表 Orders 分頁中。
  4. 角色模擬機制：藉由底部的模擬帳號面板（訪客/會員/管理員）來快速切換角色權限。會員可即時依 Email 查詢其訂餐記錄，管理員可開啟「後台管理」即時審核與變更訂單狀態（待處理、準備中、已完成、已取消）。

2. 檔案結構說明

  • index.html：核心結構。包含 Navbar、菜單列表區塊、會員訂餐紀錄面板、管理員後台面板與浮動購物車抽屜。
  • style.css：樣式表。基於 WooCommerce 經典藍色 (#046bd2) Theme，以 Flexbox 與 CSS Grid 實作響應式網頁排版。
  • script.js：前端互動邏輯。維護購物車狀態、LocalStorage 數據快取、角色切換器以及呼叫後端 API。
  • Code.gs：GAS 後端代碼。發布為 Web App 進入點，並在 Google Sheet 中自動初始化表結構 (Products / Orders)。

3. Google Apps Script 部署步驟

  1. 新建一個 Google 試算表，點選選單的「擴充功能」➡️ 「Apps Script」。
  2. 將預設代碼清空，將 Code.gs 內容複製貼上並儲存。
  3. 在 Apps Script 專案左側按「+」新增一個 HTML 檔案，命名為 index。
  4. 將專案資料夾下已經合併好的單一網頁檔案 gas_deploy.html 內容整份複製，貼上並取代預設內容儲存。
  5. 點擊右上角「部署」➡️「新增部署」，選擇「網頁應用程式」。將「執行身分」設為「我」，「誰有權存取」設為「任何人」，點選部署即可獲得線上網址。
